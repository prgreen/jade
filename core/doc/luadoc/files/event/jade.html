M.create_event (name, Name)
Create new jade events specific to your application. Events must have different names. They can be accessed with M.event.name.

Parameters

  • name:
  • Name: given to the event
M.fire_event (ev, arg)
Add an event to the event queue. Use this function to produce events.

Parameters

  • ev: A registered M.event.
  • arg: Table associated with event to transmit additional data (optional).

M.net (peer, event)
Network function for ease of integration with jade framework. Add this function with a trigger every x ticks. You must have already initiated a client or a server, so that you can use the returned enet event.

Parameters

  • peer: Local peer that will be checked for traffic (localhost).
  • event: Network event as returned by init_server or init_client functions.
M.remove_trigger (ev, func, cond)
Remove an already present trigger.

Parameters

  • ev: Event on which the function was triggered.
  • func: Function that was triggered.
  • cond: Condition function.
M.reset ()
Reset attributes for a new loop
M.run (framerate, cpu)
Main jade loop. Treats events using coroutines.

Parameters

  • framerate: Number of frames per second (default 60)
  • cpu: Used processing power, between 0 and 1 (default 60/100)
M.send_event (p, ev, arg)
Send an event through the network to a remote peer (client or server).

Parameters

  • p: Peer.
  • ev: A registered M.event.
  • arg: A table for additional data associated with the event.
M.send_event_unreliable (p, ev, arg)
Send an unreliable event through the network to a remote peer (client or server).

Parameters

  • p: Peer.
  • ev: A registered M.event.
  • arg: A table for additional data associated with the event.
M.trigger (ev, func, cond)
Trigger a function on a certain event.

Parameters

  • ev: On this event.
  • func: Trigger this function. The function will receive one table argument with the event that triggered it.
  • cond: Condition function (optional).
M.wait_for_event (ev)
Wait for specific M.event before pursuing the script. Does not block execution of other events.

Parameters

  • ev: Name of the jade event (registered with M.create_event)
M.wait_for_event_cond (ev, cond)
Wait for specific M.event and execute a condition function. If the condition is true, then the execution continues, if not the script will wait until another same event and reevaluate the condition.

Parameters

  • ev: Name of the event.
  • cond: Condition function (must return true or false).
